# testing-projects
api aqa pipeline
UI Automation (DemoQA) — автоматизированные веб-тесты на Playwright + pytest для демо-сайта с формами, модалками и drag-and-drop. Реализовано через Page Object Model.
API Testing (Reqres.in) — проверка публичного REST API: авторизация, CRUD-операции, валидация JSON-схем. Использованы requests, pytest и параметризация.
Full QA Pipeline — интеграция UI и API тестов с генерацией отчётов через Allure и автозапуском в GitHub Actions. Демонстрирует готовность к работе в CI/CD-среде.
Вместе эти проекты показывают навыки ручного и автоматизированного тестирования, работы с инструментами Agile/DevOps и умение собирать профессиональный тестовый стек «с нуля» — без коммерческого опыта.

#  QA Automation & DB Testing Demo

> **«Автоматизация — не роскошь, а способ взять на себя больше ответственности»**

Этот репозиторий — мой pet-проект как **Junior QA Engineer**, в котором я показываю, как **автоматизация тестирования** и **работа с базами данных** напрямую влияют на качество продукта и скорость команды.

В коммерческой разработке каждая минута на ручную проверку — это минута, которую нельзя потратить на исследование новых модулей, глубокий анализ требований или покрытие «серых зон» монолита.  
Я убедился: правильно настроенная автоматизация **ускоряет регрессию**, **освобождает время** и **позволяет QA брать на себя до +20–30% функционала** — даже без senior-опыта.

Здесь вы найдёте:
-  UI- и API-автотесты (Playwright + pytest)
-  Проверки через прямой запрос к базе данных (PostgreSQL / SQLite)
-  Сравнение: «ручная проверка vs автоматизированная» (в документации)
-  Чёткую структуру, документацию и примеры интеграции в CI

Этот проект — не просто демо. Это **доказательство моего подхода**:  
> *Качество — это не только «всё работает», а «мы успеваем проверить все важные дефекты и их скопления».*
